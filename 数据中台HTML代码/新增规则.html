新增规则PRD文档

1. 业务流程与交互逻辑

1.1 核心流程

用户通过四个标准步骤完成规则配置：定义基本信息与类型 -> 选择生效范围（含类型匹配校验） -> 配置规则详细参数 -> 抽样测试并保存。


2.详细功能需求

2.1 步骤一：基本信息与规则类型选择

规则名称：

控件： Input 框。校验： 必填；全局唯一；支持中英文、数字、下划线；长度 50 字符以内。


规则描述：

控件： Textarea。功能： 非必填，说明规则用途。


规则类型：

通用规则： 下拉选择（非空、唯一性、值域、枚举、长度、正则）。


自定义SQL规则： 选中后，步骤三将切换为 SQL 编辑器模式。


严重级别：

选项： 提醒 (Reminder)、警告 (Warning)、错误 (Error)。

作用： 决定质量报告中的展示颜色及是否触发下游调度熔断。


2.2 步骤二：动态参数配置

根据步骤一的类型，展示对应的输入配置：


规则子类型     配置控件     交互逻辑规范

非空校验     无     界面展示“该规则无需额外配置参数”。

唯一性校验     无     界面展示“系统将校验所选字段的全局唯一性”。

值域校验     最小值 - 最大值     两个数值输入框。需校验：最大值 ≥最小值，否则失焦报错。

枚举值校验     Textarea     允许输入枚举值，以英文逗号分隔。后端需自动过滤首尾空格。

字段长度校验     最小长度 - 最大长度     两个正整数输入框。需校验：最大长度 ≥最小长度，值必须 ≥0

正则校验     Textarea + 示例     输入 Regex Pattern。下方提供常用正则允许复制。

自定义SQL     SQL Code Editor     支持语法高亮。校验必须包含 ${TABLE_NAME} 占位符。
